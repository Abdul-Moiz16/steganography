# Notebooks Guide

The notebooks should be read as helpers around the final proposal, not the old midway design.

> Note: the notebooks may attempt deprecated behaviour. The correct entry points are the run.py or viewer.py. Documentation in the README.

## Recommended Coverage

1. ML cover generation and QC checks
2. Payload/stego manifest inspection
3. End-to-end dry-run validation
4. Metrics and plotting review

## Current Methodology Notes

- mainline detector work uses classical statistical detectors only
- use grayscale PNG/JPEG branch outputs, not a single RGB PNG carrier format
