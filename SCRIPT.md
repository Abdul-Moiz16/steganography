# VIDEO SHOOTING SCRIPT — Steganography Prototype

**Duration:** ~3 min · 6 Voices · ~30s each

---

## VOICE 1 — The Big Picture (~30s)

"So imagine you take a normal photograph and you hide secret data inside it. You're changing something about that image, right? Its statistical fingerprint shifts, just a tiny bit."

"But here's what got us excited: what if that fingerprint was already different before you even touched the image? What if an AI-generated image and a real photograph don't look the same to a detector, even before any data is hidden?"

"That's the core question behind this project. Does the origin of the carrier image, whether it's a real photo or generated by AI, change how easily we can detect hidden data inside it?"

---

## VOICE 2 — Research Questions (~30s)

"To break that down, we defined five research questions."

"The first one is the big one: if we take real photographs and ML-generated images and embed data the exact same way, do detectors behave differently? The second one zooms into the ML side, because we actually use two different generators, SDXL and FLUX, and we want to know if the choice of model matters."

"Then we go further. Does increasing the payload size widen or close that gap? Does it matter if we embed in the spatial domain versus the frequency domain? And finally, if we encrypt the payload before hiding it, does that change anything, and does it depend on where the image came from?"

"These five questions are what structure the entire experiment."

---

## VOICE 3 — Prototype Design (~30s)

"Now, we built this as two prototypes working together."

"The vertical prototype is about depth. We took each core component and implemented it fully. That means LSB embedding with configurable fill rate and bit depth, AES-256 encryption for the payload, and three statistical detectors: Chi-Square, RS Analysis, and Sample Pairs. Each one is tested in isolation, and each one works."

"The horizontal prototype is about breadth. It wires all those components into one pipeline that runs across sixty images, twenty from each source: real photos, SDXL, and FLUX. The vertical prototype proves the parts work. The horizontal prototype proves they work together, and gives us early results across all five research questions before we scale to the full experiment."

---

## VOICE 4 — Code Architecture (~30s)

"Let's look at how the code is actually organized."

"Under src, we have four packages. Acquisition handles image sourcing: it downloads real photos from COCO and generates ML images through SDXL and FLUX using the Hugging Face diffusers library. Everything gets normalized to 512-by-512 grayscale, so all carriers look identical going into the pipeline."

"Embedding is where the LSB function lives, masking and replacing the least-significant bits of selected pixels. Detection runs each statistical test independently and scores every image. Evaluation aggregates those scores into the metrics we care about: ROC-AUC, EER, and per-condition breakdowns."

"And a pipeline runner ties it all together through a single CLI command. It handles cover preparation, embedding, detection, and metrics, and writes everything into a self-contained run folder."

---

## VOICE 5 — Launching the Viewer (~30s)

"To explore the results, we built a web-based viewer. And it's really simple to get running: you install the requirements, run `python viewer.py`, and that's it. A local server starts on port 8765, you open it in your browser, and you're looking at your experiment."

"The landing page is the Runs Overview. It shows every pipeline run with its profile, configuration, and something we call the source effect delta, which is the AUC gap between real and ML-generated carriers. That number tells you at a glance whether carrier origin is making a difference."

"If we click into a run, we get the detail view. The Overview tab shows the full experiment configuration, and at the top there's a summary strip with the profile type, the experimental coverage, and that source effect value we just mentioned."

---

## VOICE 6 — Viewer Deep Dive (~30s)

"Now let's look at the tabs where the actual analysis lives."

"The Results tab is organized by research question. RQ1 shows real versus ML detectability side by side, RQ2 breaks out SDXL versus FLUX, and each card gives you per-detector AUC values with visual bars so you can immediately see the differences."

"The Gallery tab lets you dig into individual images. Every cover group shows its three source variants, and expanding a group gives you per-detector scores with bars comparing cover versus stego, so you can spot which images are easiest or hardest to detect."

"The Conditions tab gives the full factorial breakdown. Every detector row is expandable, showing EER, accuracy, and sample counts per condition. And the search bar is context-aware across the whole app: it filters runs on the overview, elements inside a run, and on the docs page it highlights keywords with navigation arrows. So wherever you are, you can find exactly what you need."
